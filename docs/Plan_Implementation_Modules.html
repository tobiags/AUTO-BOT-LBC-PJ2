Règles opérationnelles

Ces règles sont non-négociables. Elles protègent les comptes LBC, évitent les suspensions SMS, et garantissent la confidentialité des accès. Violation = incident à traiter immédiatement.

Règles MUST — obligation absolue
R01 · HORAIRES SMS

Les SMS de prospection ne partent que entre 08h00 et 20h00, heure de Paris. La tâche Celery vérifie l'heure avant chaque envoi. Un message planifié hors horaire est mis en queue pour 08h00 le lendemain.

R02 · BLACKLIST CROSS-PROJETS

Un numéro STOP reçu sur n'importe quelle SIM est blacklisté dans les deux projets P1 et P2 simultanément. La table blacklist est partagée — aucune exception.

R03 · ACCÈS VIA BITWARDEN

Toutes les clés API, mots de passe, et tokens sont stockés et partagés exclusivement via Bitwarden. Zéro envoi par email, SMS, WhatsApp, ou GitHub issue. Aucune clé dans le code.

R04 · WARM-UP OBLIGATOIRE

Tout compte LBC nouvellement créé reste en statut EN_CHAUFFE pendant 48–72h minimum (0 message envoyé, navigation simulée uniquement). Aucune campagne ne peut cibler un compte EN_CHAUFFE. Passage à ACTIF uniquement si datadome_trust_level ≥ MEDIUM.

R05 · QUOTAS PROGRESSIFS

SMS SIMs SMSTools : 15/SIM/jour au démarrage, montée par paliers après validation (cible 30–60/SIM/jour, soit 300–500 SMS/jour sur 8 SIMs). Messagerie comptes LBC : 10/compte/jour S1, +5/semaine, plafond 30/compte/jour. Délai entre messages : 2 à 12 minutes aléatoire (jamais de précision milliseconde).

R06 · SNAPSHOTS AVANT DÉPLOIEMENT

Créer un snapshot Hetzner avant tout déploiement majeur (migration DB, mise à jour Patchright, changement de quota). Le snapshot doit exister avant le merge sur main.

Règles NEVER — interdiction absolue
R07 · JAMAIS IP DATACENTER POUR LBC

Ne jamais créer un compte LeBonCoin avec l'IP du VPS Hetzner ou un proxy datacenter. DataDome classe l'AS Hetzner comme suspect. Utiliser exclusivement iproxy.online (4G FR).

R08 · JAMAIS LE DOMAINE OFFICIEL

Ne jamais utiliser le domaine officiel AutoTransfert pour les emails LBC automatisés. Utiliser exclusivement le domaine opérationnel dédié (reprise-auto-pro.fr ou équivalent).

R09 · JAMAIS DE SÉLECTEURS CSS HARDCODÉS

Ne jamais piloter LeBonCoin avec des sélecteurs CSS ou XPath fixes dans le code. LBC change son HTML régulièrement. Utiliser browser-use Cloud (IA) ou crawl4ai (extraction sémantique).

R10 · JAMAIS MOCK EN DEHORS DE BOUNDARIES

Ne jamais mocker PostgreSQL, Redis, ou les services internes dans les tests. Mocker uniquement boundaries.py (les APIs externes). Les tests utilisent une vraie DB de test.

Règles ALWAYS — discipline quotidienne
R11 · SENTRY AVANT PRODUCTION

Sentry doit être configuré (DSN Python + DSN JS) avant la mise en production. Toute exception non gérée doit créer une alerte Sentry. Ne jamais shipper sans monitoring.

R12 · IDEMPOTENCE DES WEBHOOKS

Tous les endpoints webhook (/webhooks/sms, /webhooks/email, /webhooks/call) doivent être idempotents. Mailgun et SMSTools peuvent livrer le même événement deux fois. Clé d'unicité en DB.

Workflows opérationnels

Cinq workflows couvrent 100% des opérations du système. Chaque étape indique qui l'exécute (API/DB/Externe) et ce qui se passe en cas d'erreur.

WF-01 — Création d'un compte LeBonCoin
API
Déclenchement Celery
Tâche create_lbc_account placée en queue. Vérifie le pool de comptes : minimum 3 ACTIFS à maintenir en permanence. Création déclenchée si pool descend sous le seuil ou si un compte passe en QUARANTAINE.
EXT
Obtenir IP 4G française — iproxy.online
Appel rotate_4g_ip() puis get_4g_proxy(). Vérification ASN (doit être Orange/SFR/Bouygues). Erreur → ProxyUnavailableError, arrêt
boundaries.rotate_4g_ip() → boundaries.get_4g_proxy()
EXT
Générer email unique — Mailgun
Appel generate_email(domain) → contact.{hex8}@reprise-auto-pro.fr. Sauvegarde en DB avec statut PENDING.
email = generate_email(settings.operational_domain)
EXT
Acheter numéro OTP — SmsApp.io
Appel buy_number("france","leboncoin"). Retourne {id, phone, cost}. Erreur → NoStockError, retry 60s
order = await boundaries.buy_number("france","leboncoin")
EXT
Naviguer + créer compte — Mode A : Patchright + iproxy 4G
Mode A (principal) : Patchright (Chrome patché anti-détection) routé via le proxy 4G FR iproxy. Navigation LBC → formulaire inscription → email + téléphone OTP. Délais 3–8s, scroll humain. Fallback Mode B : si iproxy indisponible, browser-use Cloud avec IP résidentielle FR (trust score légèrement inférieur, warm-up plus long).
EXT
Attendre code SMS — poll SmsApp.io
poll_sms(order.id, max_wait=120) toutes les 3s. Timeout → cancel_number() → remboursement → retry
code = await boundaries.poll_sms(order["id"], max_wait=120)
EXT
Attendre code email — webhook Mailgun
LBC envoie un lien de confirmation email. Mailgun route vers POST /webhooks/email. extract_verification_code(body) extrait le code.
DB
Sauvegarder session + cookies DataDome
Compte créé → statut EN_CHAUFFE en DB. Cookies DataDome chiffrés. Timer 48–72h démarré. datadome_trust_level=LOW initial.
db.account.update(id, status="EN_CHAUFFE", cookies=encrypted_cookies, datadome_trust_level="LOW")
OK
Fin — compte prêt en 48–72h
Alerte Sentry si échec. Prochaine tâche Celery : après 48–72h de navigation simulée, si datadome_trust_level ≥ MEDIUM → passage ACTIF. Sinon, prolongation warm-up.
WF-02 — Envoi d'une campagne SMS (P1 ou P2)
API
POST /campaigns — création campagne
Payload : {type, targets[], message_template, quota_per_sim}. Validation Pydantic. Statut PENDING en DB.
API
Vérification horaire (R01)
Si heure < 08h00 ou > 20h00 → message planifié pour 08h00. Celery beat gère le scheduling.
DB
Filtrage blacklist (R02)
JOIN targets avec blacklist. Exclusion des numéros STOP. Log du nombre d'exclusions.
SELECT t.phone FROM targets t WHERE t.phone NOT IN (SELECT phone FROM blacklist)
EXT
Sélection SIM — round-robin pondéré + délai inter-SMS
Choisir la SIM ACTIVE avec le quota restant le plus élevé. Exclure SIMs RALENTIE ou BLOQUÉE. Délai aléatoire 2–12 min depuis le dernier SMS de cette SIM (R05 — anti-pattern milliseconde).
sim = select_best_sim(available_sims, daily_quotas); await sleep(uniform(120, 720))
EXT
Envoi SMS — boundaries.send_sms()
Appel SMSTools. 402 → InsufficientCreditError → stopper campagne, alerte. Retry x2 sur timeout réseau. Le corps doit contenir "STOP au XXXX pour ne plus recevoir de SMS" (LCEN).
result = await boundaries.send_sms(sim.id, to=phone, body=rendered_message)
DB
Enregistrement résultat
Statut SENT ou FAILED. Incrémentation quota SIM du jour. Timestamp pour respect fenêtre horaire.
OK
Fin — dashboard mis à jour en temps réel
WebSocket push vers le back-office. Compteurs campagne mis à jour.
WF-03 — Réception d'un appel entrant (vendeur rappelle)
EXT
Webhook SMSTools → POST /webhooks/call
Payload : {sim_id, from: "+33X...", timestamp}. Simultanément, SMSTools forwardl'appel vers le téléphone client.
DB
Lookup annonce associée
get_listing_by_sim_and_phone(sim_id, caller) → recherche le dernier SMS envoyé depuis cette SIM vers ce numéro.
WS
Push WebSocket back-office
Broadcast {event:"incoming_call", caller, listing:{url,title,price,vehicle}}. Le collaborateur voit l'annonce avant de décrocher.
OK
Log appel en DB
Enregistrement horodaté. Champ outcome à remplir manuellement dans le back-office après la conversation.
WF-04 — Scraping annonces LBC (tâche quotidienne)
SCH
Celery beat — déclenchement 06h00
Tâche scrape_lbc_listings et scrape_lacentrale_listings lancées en parallèle.
EXT
Navigation LBC — Patchright (session compte ACTIF) ; La Centrale — crawl4ai direct
LBC : session Patchright avec cookies DataDome rechargés depuis DB (compte ACTIF du pool). La Centrale : crawl4ai direct (pas de DataDome). crawl4ai/Scrapling extraient titre, prix, km, téléphone, url en format unifié.
AI
Extraction numéro — regex puis Claude Haiku
extract_phone(text) : regex d'abord. Si ambiguïté (ex: "appel le 06 XX XX XX XX") → extract_phone_llm(text) via Claude Haiku. ~0.001€/appel

Déduplication + sauvegarde
Upsert sur listing_url. Marquage NOUVELLE pour les annonces jamais vues. Filtre blacklist immédiat.

Annonces disponibles pour campagne
Statut NOUVELLE → prêtes pour WF-02. Dashboard mis à jour.
WF-05 — Réception STOP (désabonnement)
EXT
SMS entrant "STOP" → POST /webhooks/sms
SMSTools envoie le webhook immédiatement. Normalisation du corps : STOP / ARRET / DESABONNEMENT.

Ajout en blacklist cross-projets (R02)

Confirmation SMS automatique

03

iproxy.online — Proxy 4G mobile

IP opérateur français (Orange/SFR/Bouygues/Free Mobile) pour la création de comptes LBC uniquement (Mode A). Inactif le reste du temps — pay-per-GB. ~15–30 €/mois.

📡
Rôle
Activation ponctuelle · SIM physique en rack
CRÉATION COMPTES LBC — jamais pour autre chose (R07)

DataDome maintient un modèle ML spécifique pour LBC. Une IP datacenter (ASN Hetzner) est bloquée immédiatement. L'IP 4G d'un opérateur français passe les vérifications d'ASN, de géolocalisation, et de réputation.

Règle R07 : ne jamais utiliser l'IP du VPS Hetzner pour créer un compte LBC. Si iproxy est indisponible, stopper la création — ne pas contourner.

Checklist avant création de compte LBC OPÉRATIONS

  • iproxy.online actif — vérifier via GET /v1/proxy/active avant de démarrer
  • IP vérifiée comme FR opérateur — contrôle ASN via ipinfo.io (Orange/SFR/Bouygues)
  • Rotation effectuée — rotate_4g_ip() appelé depuis la dernière création (délai 30–60s)
  • Slot compte disponible — moins de 8 comptes EN_ACTIF ou EN_CHAUFFE
  • Numéro OTP disponible — SmsApp.io stock France LeBonCoin vérifié
  • Domaine email opérationnel — DNS Mailgun résout correctement

False  // rotation échouée → lever ProxyUnavailableError
